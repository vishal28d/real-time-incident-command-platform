# Incident Management Platform - Enhancement Plan

## Project Overview
Strengthen the existing incident management platform to be interview-ready with comprehensive features demonstrating production-grade skills.

---

## 1. Observability & Monitoring

### 1.1 Add Spring Boot Actuator
- Add `spring-boot-starter-actuator` dependency to all services
- Expose `/actuator/health`, `/actuator/metrics`, `/actuator/info`
- Configure health indicators for Kafka and MongoDB

### 1.2 Prometheus Metrics
- Add `micrometer-registry-prometheus` to incident-service
- Expose `/actuator/prometheus` endpoint
- Create custom metrics:
  - `incident.created.count` - incidents created per time window
  - `incident.resolved.count` - incidents resolved per time window
  - `alert.ingested.count` - alerts ingested

### 1.3 Distributed Tracing Setup
- Add OpenTelemetry Spring Boot starter
- Configure trace export to Jaeger (optional local)

---

## 2. Resilience Patterns

### 2.1 Circuit Breaker
- Add Resilience4j `spring-circuitbreaker` to all services
- Configure circuit breaker for:
  - Kafka producer calls
  - MongoDB operations
  - Inter-service HTTP calls

### 2.2 Retry with Backoff
- Configure Resilience4j retry for:
  - Kafka publish failures (3 retries with exponential backoff)
  - MongoDB connection failures
- Add `@Retryable` annotations

### 2.3 Dead Letter Queue
- Configure Kafka DLQ for failed event processing
- Add DLQ consumer endpoint to inspect failed events
- Add retry mechanism for DLQ events

### 2.4 Graceful Degradation
- Add fallback responses when services unavailable
- Cache recent incidents for offline access

---

## 3. Full-Stack Features

### 3.1 Incident Assignment
- Add `assignedTo` field to Incident model
- Add `PUT /api/v1/incidents/{id}/assign` endpoint
- List assigned incidents per user

### 3.2 Comments/Notes
- Create `IncidentComment` model with:
  - author, content, createdAt, incidentId
- Add CRUD endpoints for comments
- Real-time comment updates via SSE

### 3.3 Tags & Categories
- Add `tags` field (list of strings) to Incident
- Add endpoints to add/remove tags
- Filter incidents by tag

### 3.4 Search & Filtering
- Add search query parameter to list incidents
- Filter by:
  - severity (CRITICAL, HIGH, MEDIUM, LOW)
  - status (OPEN, RESOLVED)
  - serviceName
  - assignedTo
  - date range

### 3.5 Incident Timeline
- Add timeline events model
- Track all state changes with timestamps
- Show complete history per incident

## 5. Security (Optional)

### 5.2 Rate Limiting
- Add rate limiting for ingestion endpoint

## Implementation Order

1. **Observability** (lowest effort, high impact)
   - Add actuator + prometheus to incident-service
   - Add custom metrics

2. **Full-Stack Features** (core business logic)
   - Add Incident fields (assignedTo, tags)
   - Add Comment model and endpoints
   - Add search/filter

3. **Resilience** (production-ready)
   - Add circuit breaker
   - Configure retry + DLQ

4. **API Docs** (developer experience)
   - Add OpenAPI
